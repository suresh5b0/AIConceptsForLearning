What is Generative AI?

Generative AI creates new content like text, images, audio, and code. It uses models trained on large datasets to produce creative outputs.

Large Language Models (LLMs)

LLMs are AI models trained to predict and generate language. They power chatbots, summarization, translation, and knowledge retrieval.

Transformers

Transformers use attention to process sequences in parallel. They are the foundation for modern LLMs and multimodal models.

Tokens and Embeddings

Text is broken into tokens; embeddings convert tokens into numeric vectors that capture meaning and relationships.

Prompt Engineering

Prompt engineering designs the instructions given to AI. Good prompts make the model output more accurate, useful, and safe.

Retrieval-Augmented Generation (RAG)

RAG combines retrieval from documents with generation. It helps models answer questions using up-to-date or domain-specific knowledge.

Fine-Tuning

Fine-tuning adapts a pre-trained model to a specialized task or domain. This improves performance on business-specific data.

AI Safety and Hallucinations

Safety focuses on reducing bias, errors, and harmful outputs. Hallucinations are when models generate plausible but incorrect information.

LLMs

Large Language Models

What are LLMs?

LLMs are AI models trained on huge amounts of text so they can understand questions, write text, summarize content, and help with many language tasks.

How LLMs work

They learn patterns in language and predict the next word or token. This helps them generate useful responses, explanations, and code.

Why do we use LLMs?

We use LLMs because they can save time, automate repetitive writing tasks, support customer service, help developers, and make information easier to search.

Simple examples

Example: an LLM can summarize a long report, answer a question about company policy, or generate a Python function from a request.

Why LLMs are useful

  • They understand natural language, so people can interact with them easily.
  • They can help with writing, learning, coding, and brainstorming.
  • They can be connected to tools, documents, and workflows for smarter applications.
  • They can be used in chatbots, assistants, search systems, and automation tools.

How it works when you type a query in a chat window

When a user sends a message, the system follows a short flow to produce a response.

  1. Read the question: The model receives your text input.
  2. Understand the intent: It tries to understand what you want.
  3. Use its training and context: It uses patterns learned during training and any recent chat context.
  4. Generate an answer: It predicts a helpful response word by word.
  5. Return the reply: The final answer is shown in the chat window.

In short: question → understanding → response generation → chat reply.

Easy examples of LLM usage

  • Student: Ask for a simple explanation of neural networks.
  • Developer: Ask for a Python script to read a CSV file.
  • Business: Ask for a summary of meeting notes and action items.
  • Support team: Use an LLM to answer common questions from documentation.

RAG

Retrieval-Augmented Generation

What is RAG?

RAG enriches generation with retrieved context from documents or knowledge stores.

Why use RAG?

It reduces hallucinations and improves relevance for factual, up-to-date answers.

RAG Step-by-Step

RAG works in simple stages: bring in knowledge, find the right pieces, and use them to answer.

  1. Data ingestion: Collect documents such as PDFs, websites, notes, or database records.
  2. Clean and organize: Remove irrelevant text, fix formatting, and prepare the content.
  3. Split into chunks: Break long documents into smaller pieces so retrieval is more accurate.
  4. Create embeddings: Turn each chunk into a vector using an embedding model.
  5. Store in a vector database: Save the vectors along with useful metadata like source and title.
  6. Receive a user question: The user asks something that needs an answer.
  7. Find similar content: Convert the question into a vector and search the database for the closest matches.
  8. Send context to the LLM: Give the retrieved chunks to the model as extra context.
  9. Generate the final answer: The LLM answers using the retrieved information and can cite the source.
  10. Improve over time: Review results, adjust chunk size, and refine retrieval quality.

Vector Databases and Retrieval

Vector databases store embeddings and support fast similarity search for semantic retrieval, FAQ bots, and knowledge assistants.

  • Why they matter: They let you search by meaning rather than exact keywords.
  • Common options: Pinecone, Weaviate, Milvus, Chroma, pgvector, and Azure AI Search.
  • Key features: Metadata filters, hybrid search, reranking, and approximate nearest neighbor indexes.
  • Best practice: Keep chunk size small, store useful metadata, and refresh embeddings when content changes.

RAG Approach in Practice

A production RAG system usually combines retrieval, ranking, prompt construction, and evaluation.

  • Simple RAG: Retrieve top-k chunks and append them to the prompt.
  • Advanced RAG: Add reranking, citation checks, chunk metadata, and query rewriting.
  • Agentic RAG: Let an agent decide when to search, which tools to use, and how to verify results.

RAG Example

Use retrieved documents with a prompt to generate an accurate answer.

query = "How do I set up MFA for my account?"
docs = vector_store.similarity_search(query, k=3)
context = "\n\n".join([doc.page_content for doc in docs])
prompt = f"Use the information below to answer the question:\n\n{context}\n\nQuestion: {query}"
response = llm.generate(prompt)

Realtime RAG

Realtime RAG updates retrieval results as new documents arrive and can stream answers while keeping them grounded.

  • Use live ingestion for current documents and logs.
  • Refresh embeddings automatically on data changes.
  • Stream partial answers to users while retrieval continues.
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class QueryRequest(BaseModel):
    query: str

@app.post("/query")
def query_knowledge(body: QueryRequest):
    docs = vector_store.search(body.query, top_k=4)
    context = "\n\n".join([doc.text for doc in docs])
    prompt = f"Use the facts below to answer the question:\n\n{context}\n\nQuestion: {body.query}"
    answer = llm.generate(prompt)
    return {"answer": answer}

Realtime RAG Use Cases

  • Customer support: Answers questions using the latest help articles and product updates. Why RAG? It gives fresh, accurate answers without retraining the model.
  • Operations monitoring: Explains alerts and logs from live systems. Why RAG? It pulls current incident details and helps teams respond faster.
  • Sales assistant: Finds current pricing, inventory, and policy information. Why RAG? It ensures the assistant uses the latest business data.
  • HR chatbot: Answers policy questions from employee handbooks and internal documents. Why RAG? It keeps responses grounded in approved company knowledge.
  • Legal or medical search: Retrieves relevant documents for review. Why RAG? It improves accuracy by grounding answers in trusted sources.

Why we use RAG in real-time systems

RAG is useful when the answer must be based on current or private information.

  • It reduces hallucinations by giving the model real retrieved context.
  • It allows updates without retraining the model every time content changes.
  • It works well for chatbots, search tools, support systems, and internal knowledge assistants.

RAG vs Fine-Tune Comparison

Aspect RAG (Retrieval-Augmented Generation) Fine-Tuning
How it works Retrieves external documents at runtime and passes them as context to the model. Modifies model weights offline using labeled training data.
Model changes Model stays unchanged; only context is added. Model weights and parameters are permanently updated.
Update frequency Can add new documents anytime without retraining. Requires retraining to incorporate new knowledge.
Cost Lower cost (vector storage + retrieval). Higher cost (compute, GPU time, data labeling).
Latency Slightly higher (retrieval + generation). Lower (only generation phase).
Data needed Documents, no labeling required. Labeled input/output pairs.
Best for Up-to-date knowledge, large document sets, Q&A systems. Task-specific style, consistent behavior, specialized domains.
Hallucination risk Lower (grounds answers in retrieved docs). Higher (model may generate plausible but false outputs).

When to Use RAG

  • You have a large knowledge base that changes frequently.
  • You need factual accuracy and want to reduce hallucinations.
  • You want minimal computational overhead and faster deployment.
  • Your documents are already available (no labeling needed).

When to Use Fine-Tuning

  • You need a specialized model for a specific task or domain.
  • You have labeled training data and consistent patterns.
  • You need lower latency or offline inference.
  • You want the model to learn a particular response style or format.

How to Fine-Tune

Fine-tuning adapts an existing model using labeled examples or a domain-specific dataset.

In simple words: start with a strong pre-trained model, teach it with examples, and then use the updated version for your task.

  • Step 1: Choose a base model that matches your task, such as a text generation or classification model.
  • Step 2: Collect and format training data with input/output pairs, such as CSV, JSONL, or a dataset object.
  • Step 3: Tokenize your data with the model tokenizer and prepare training and validation splits.
  • Step 4: Set training settings like batch size, learning rate, number of epochs, and evaluation strategy.
  • Step 5: Train the model using a Trainer or fine-tuning API, then test it on sample examples.
  • Step 6: Save the fine-tuned model, upload it to a model hub, or deploy it in an app.

What changes after fine-tuning? The model becomes better at your specific style, task, or domain, but it does not automatically learn new facts unless you retrain it with updated data.

Important note: Fine-tuning needs good quality data. If the examples are noisy or inconsistent, the model may learn the wrong pattern.

from datasets import load_dataset
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM, TrainingArguments, Trainer

model_name = "google/flan-t5-base"
dataset = load_dataset("csv", data_files={"train": "data/train.csv", "validation": "data/valid.csv"})

tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

def preprocess(example):
    inputs = tokenizer(example["input_text"], truncation=True, padding="max_length", max_length=256)
    targets = tokenizer(example["target_text"], truncation=True, padding="max_length", max_length=128)
    inputs["labels"] = targets["input_ids"]
    return inputs

dataset = dataset.map(preprocess, batched=True)

training_args = TrainingArguments(
    output_dir="./fine-tuned-model",
    evaluation_strategy="epoch",
    learning_rate=2e-5,
    per_device_train_batch_size=8,
    per_device_eval_batch_size=8,
    num_train_epochs=3,
    save_total_limit=2,
    load_best_model_at_end=True,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    eval_dataset=dataset["validation"],
)

trainer.train()
trainer.save_model("./fine-tuned-model")

Prompt Engineering

Prompt Engineering Guide

What Is a Prompt?

A prompt is the input you give to an AI model.

Why Prompt Engineering Matters

LLMs generate outputs based on the instructions they receive.

Think of prompting like giving directions:

  • Vague directions → unclear result
  • Clear directions → better result
Prompt:
Write an essay.

Result:

  • Unclear topic
  • Unclear length
Write a 300-word essay about climate change for high school students.

Result:

  • Specific topic
  • Specific audience
  • Specific length

3. Anatomy of a Good Prompt

A strong prompt often contains:

1. Role

You are an experienced AI teacher.

2. Task

Explain neural networks.

3. Context

The audience is beginners who know basic Python.

4. Format

Use bullet points and examples.

Formula:

Role + Task + Context + Format
You are an AI teacher.

Explain neural networks to beginners.

Assume they know basic mathematics.

Use simple language and examples.

4. Zero-Shot Prompting

The model receives instructions without examples.

Translate this sentence into Spanish:
I love learning AI.

This is called: Zero-shot Learning.

5. One-Shot Prompting

You provide one example.

Input: Dog
Output: Animal

Input: Rose
Output:

The model learns the pattern from the example.

6. Few-Shot Prompting

You provide several examples.

Input: Dog
Output: Animal

Input: Rose
Output: Plant

Input: Eagle
Output:

This technique often improves reliability.

7. Chain-of-Thought Prompting

Solve this math problem step by step.

This often improves reasoning quality and is useful for learning.

8. Structured Prompting

Explain machine learning using:

1. Definition
2. Example
3. Advantages
4. Disadvantages

The output becomes easier to read.

9. Constraint-Based Prompting

Explain AI in fewer than 100 words.
Use language suitable for a 12-year-old.

Constraints help control outputs.

10. Prompt Chaining

Break a large task into smaller prompts.

Step 1:
Generate startup ideas.

Step 2:
Select the best idea.

Step 3:
Create a business plan.

This is commonly used in AI workflows and agents.

11. Prompting for Coding

Write a Python function that finds the largest number in a list.

Requirements:
- Use type hints
- Add comments
- Include an example

Specific prompts generally produce better code.

12. Prompting for Agentic AI

Goal:
Plan a 3-day trip to Hyderabad.

Constraints:
Budget under ₹15,000.

Output:
Daily itinerary and estimated costs.

The agent can then plan, search, use tools, and refine results.

13. Common Prompting Mistakes

Too Vague

Teach AI.
Teach the basics of machine learning to a beginner.

Missing Context

Explain transformers.
Explain transformers to a student learning Generative AI for the first time.

Missing Output Format

Summarize this article.
Summarize this article in 5 bullet points.

Task:

Output Format:
Markdown

Agent Fundamentals

Understand the core components: LLMs as the brain, memory for state, and tools for actions.

Planning & Reasoning

Learn how agents break down tasks, create action plans, and reason through problems.

Tool Integration

Connect agents to external systems (APIs, databases, web search) to perform real-world operations.

Memory Systems

Implement short-term and long-term memory to maintain context and learn from past interactions.

Multi-Agent Collaboration

Explore how multiple agents with specialized roles can work together to solve larger problems.

Evaluation & Safety

Develop strategies to evaluate agent performance, ensure reliability, and mitigate risks.

AI Agent Learning Path

  1. Understand how LLMs work and how to write clear prompts.
  2. Build a basic chatbot and define clear agent goals.
  3. Add retrieval from a vector database for grounded answers.
  4. Connect simple tools like search, calculators, or document lookup.
  5. Use short-term memory to keep context across turns.
  6. Test the agent with real examples and refine its decisions.

AI Agents in Practice

  • Customer Service Agent: Handles inquiries, accesses FAQs, and escalates complex issues.
  • Data Analyst Agent: Queries databases, performs calculations, and generates reports.
  • Software Development Agent: Writes code, runs tests, and fixes bugs based on requirements.
  • Personal Assistant Agent: Manages schedules, sends emails, and provides recommendations.
  • Research Agent: Gathers information from various sources, summarizes findings, and identifies key insights.

Phase 3

Agentic AI

Agentic AI is the next step beyond simple agents: it combines planning, memory, tool use, and multi-agent orchestration to solve more complex, real-world problems. It emphasizes autonomous decision-making and adaptive behavior.

Autonomous Decision-Making

Agents independently decide actions based on goals, observations, and available tools, adapting to dynamic environments.

Adaptive Learning & Memory

Agents learn from past interactions, store relevant information, and retrieve knowledge to improve future performance and decision-making.

Complex Task Orchestration

Break down intricate problems into manageable sub-tasks, coordinate actions, and synthesize results from multiple tools or sub-agents.

Tool Integration & Extension

Seamlessly incorporate diverse external tools (APIs, databases, web search) to expand capabilities and interact with the real world.

Proactive Problem Solving

Agents can anticipate issues, proactively seek solutions, and execute multi-step plans without constant human intervention.

Human-in-the-Loop Design

Implement mechanisms for human oversight, feedback, and intervention to ensure safety, ethical alignment, and performance tuning.

Agentic AI Learning Track

  1. Build a simple LLM-based assistant first.
  2. Add retrieval from documents or a knowledge base.
  3. Integrate one tool, such as search or a calculator.
  4. Add memory for user preferences and recent tasks.
  5. Enable the agent to plan and revise multiple actions.
  6. Experiment with multiple agents collaborating on a shared goal.

Advanced Agentic Concepts

  • Self-Correction: Agents identify and rectify their own mistakes or inefficient actions.
  • Multi-Agent Systems (MAS): Designing and orchestrating teams of agents with specialized roles to achieve common goals.
  • Cognitive Architectures: Building agents with human-like cognitive functions like perception, attention, and reasoning.
  • Emotional AI: Developing agents that can understand, interpret, and respond to human emotions (ethically and safely).
  • Emergent Behavior: Observing complex, unprogrammed behaviors arising from simple agent interactions within a system.
# Agentic AI pseudocode
user_goal = "Plan a training session on RAG and LLMs"
plan = planner.create_plan(user_goal)
steps = plan.break_into_steps()
results = []
for step in steps:
    if step.requires_search:
        data = search_tool.run(step.query)
    elif step.requires_math:
        data = calculator.run(step.expression)
    else:
        data = llm.generate(step.prompt)
    results.append(data)
summary = llm.generate("Combine these step results into a learning plan:")
print(summary)

Agentic AI Example Use Cases

  • Learning coach: builds a study plan, gathers resources, and explains concepts step by step.
  • Research assistant: collects evidence from papers, summarizes findings, and generates recommendations.
  • Automation agent: reads system status, executes remediation tools, and reports updates.
  • Customer support flow: auto-triages requests, consults knowledge bases, and suggests resolutions.

Agentic AI Learner Tips

  • Focus on one improvement at a time: prompt design, then retrieval, then tool use.
  • Use trace logs to inspect how the agent decides and what tools it calls.
  • Keep prompt templates separate from system logic for easier tuning.
  • Start with constrained tasks before opening up broader agent freedom.
  • Measure success by correctness, tool efficiency, and user satisfaction.

FastMCP lets you expose AI tools and context-aware model behavior through a lightweight API interface.

from fastmcp import FastMCP, Tool
from fastapi import FastAPI

app = FastAPI()

@mcp.tool
def search_docs(query: str) -> str:
    return "Results for: " + query

@app.post("/mcp")
def handle_mcp(request: dict):
    return mcp.dispatch(request)

LangChain: Build AI Applications

LangChain is a framework for developing applications powered by language models. It connects prompts, chains, memory, tools, and data sources into production-ready AI workflows.

⛓️

Chains

Combine prompts and LLMs into sequential workflows for complex reasoning and multi-step tasks.

🔧

Tools & Agents

Equip AI with access to external tools, APIs, and services for autonomous decision-making.

💾

Memory

Maintain conversation context and conversation history across multiple interactions.

📚

RAG

Retrieve documents and augment generation with domain-specific knowledge.

Core Features

  • LLM Support: OpenAI, Google, Anthropic, Ollama, Hugging Face
  • Prompt Templates: Reusable, parameterized prompts
  • Output Parsing: Structure and extract data from LLM outputs
  • Vector Stores: Pinecone, Weaviate, Milvus, Chroma
  • Memory Management: Conversation history and context windows
  • Tool Integration: @tool decorator for function as tools
  • Async Support: Non-blocking I/O for scalability

Quick Start

from langchain_ollama import OllamaLLM
from langchain_core.prompts import PromptTemplate

# Initialize model
llm = OllamaLLM(model="mistral")

# Create prompt template
prompt = PromptTemplate(
    input_variables=["topic"],
    template="Explain {topic} briefly"
)

# Build chain
chain = prompt | llm

# Run chain
response = chain.invoke(
    {"topic": "machine learning"}
)
print(response)

from langchain_core.tools import tool
from langchain.agents import create_react_agent, AgentExecutor
from langchain import hub

# Define tools
@tool
def calculate(expression: str) -> float:
    """Calculate mathematical expressions"""
    return eval(expression)

@tool
def get_info(topic: str) -> str:
    """Get information about a topic"""
    return f"Information about {topic}..."

# Create agent

    {"input": "What is 25 * 4? And explain quantum computing."}

LangGraph: Visual Workflow Design

LangGraph is a library for building stateful, multi-actor applications with LLMs. It extends LangChain with graph-based state management and deterministic workflows.

📊

Graph-Based States

Model workflows as directed graphs with nodes representing operations and edges representing state transitions.

🔄

Cyclic Workflows

Support feedback loops, retries, and iterative refinement within agent workflows.

🎯

Deterministic Execution

Replay, debug, and trace exact execution paths through complex agent logic.

🔍

Visualization

See your workflow as a visual graph during development and debugging.

Key Concepts

  • State: Shared data structure passed between nodes
  • Node: Function that processes state and returns updates
  • Edge: Connection between nodes with routing logic
  • Conditional Edges: Dynamic routing based on state
  • Cycles: Feedback loops for iterative refinement
  • Checkpointing: Save state snapshots for debugging
  • Streaming: Real-time output as graph executes

Simple Example

from langgraph.graph import StateGraph
from typing_extensions import TypedDict

# Define state
class AgentState(TypedDict):
    messages: list
    question: str
    answer: str

# Create graph
graph = StateGraph(AgentState)

# Define nodes
def query_node(state):
    # Process question
    return {"answer": "Processing..."}

def refine_node(state):
    # Refine answer
    return {"answer": "Refined answer"}

# Add nodes
graph.add_node("query", query_node)
graph.add_node("refine", refine_node)

# Add edges
graph.add_edge("query", "refine")
graph.set_entry_point("query")
graph.set_finish_point("refine")

# Compile and run
app = graph.compile()
result = app.invoke(
    {"messages": [], "question": "What is AI?"}
)

Advanced: Conditional Routing

Make workflow decisions based on state:

def should_refine(state) -> bool:
    """Check if answer needs refinement"""
    return len(state["answer"]) < 50

# Add conditional edge
graph.add_conditional_edges(
    "query",
    should_refine,
    {
        True: "refine",     # If refinement needed
        False: "complete"   # Otherwise complete
    }
)

# This creates a dynamic workflow that adapts based on state

Model Context Protocol

MCP Overview

MCP makes it easier to connect AI models and clients to external tools safely, reliably, and with consistent behavior.

Why MCP?

  • Standardize tool interfaces
  • Enable secure external access
  • Support reusable integrations
  • Simplify multi-client workflows

When to Use MCP

  • When AI needs external tool access
  • When multiple apps use the same tool set
  • When security and observability matter
  • When you want a protocol-driven agent ecosystem

MCP Inspector Tool

The MCP Inspector helps teams review registered tools, validate request and response schemas, and trace tool call lifecycle events.

  • Inspect tool manifests and supported operations.
  • Validate input/output schemas before deployment.
  • Replay tool calls using sample payloads.
  • Debug authorization, routing, and response formats.

How MCP Works

Model Context Protocol defines how a model client discovers tools, exchanges structured tool requests, and receives tool results in a consistent format.

  • Define tool capabilities with metadata, input schema, and output schema.
  • Register tools with a protocol server or gateway.
  • Send model-generated tool calls through the MCP connector.
  • Return tool outputs as structured context for the model to continue the workflow.

MCP Configuration Example

Here is a simple tool manifest and client configuration pattern for MCP.

{
  "tool_name": "weather_service",
  "description": "Fetch current weather and guidance for a city.",
  "endpoint": "https://api.example.com/weather",
  "auth": {
    "type": "bearer",
    "token_env": "WEATHER_API_KEY"
  },
  "inputs": {
    "city": "string"
  },
  "outputs": {
    "temperature": "number",
    "condition": "string",
    "advice": "string"
  }
}
# Python example using MCP client configuration
from mcp import MCPClient

client = MCPClient(
    model="gpt-4o-mini",
    tools=["weather_service"],
    config={
        "weather_service": {
            "endpoint": "https://api.example.com/weather",
            "auth_token": os.environ.get("WEATHER_API_KEY")
        }
    }
)

response = client.run("Get current weather for Seattle and tell me whether to bring an umbrella.")
print(response)

Key MCP Concepts

  • Tool manifest: Describes the name, inputs, outputs, and auth requirements.
  • Protocol server: Routes tool calls and applies security rules.
  • Client configuration: Connects the AI model to the registered MCP tools.
  • Secure credentials: Use environment variables or secrets managers to store tokens.
